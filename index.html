    Programmation Orientée Objet (POO) - Résumé

    Introduction

    La Programmation Orientée Objet (POO) est un paradigme de programmation basé sur l'organisation du code en objets. Chaque objet est une instance d'une classe, qui est un modèle définissant les attributs (données) et les comportements (méthodes) associés à cet objet.

    Principes de Base

    1. Classe et Objet

    Une classe est un modèle pour créer des objets. Elle définit les attributs et méthodes que les objets créés à partir de cette classe posséderont.

    Un objet est une instance d'une classe. Il possède ses propres valeurs pour les attributs définis par sa classe.

    Exemple :

    class Voiture {
      String couleur;
      void conduire() { ... }
    }
    Voiture maVoiture = new Voiture();

    2. Encapsulation

    L'encapsulation consiste à restreindre l'accès direct aux attributs d'une classe et à fournir des méthodes pour les manipuler. Elle permet de protéger les données en les rendant accessibles uniquement via des méthodes spécifiques (getters et setters).

    Exemple :

    class Personne {
      private String nom;
      public String getNom() { return nom; }
      public void setNom(String nom) { this.nom = nom; }
    }

    3. Héritage

    L'héritage est le mécanisme par lequel une classe peut hériter des attributs et des méthodes d'une autre classe. Cela permet de créer des relations de type "est un" et de réutiliser le code.

    Exemple :

    class Animal {
      void manger() { ... }
    }
    class Chien extends Animal {
      void aboyer() { ... }
    }

    4. Polymorphisme

    Le polymorphisme permet d'utiliser une méthode de manière différente selon les objets. Il existe deux types de polymorphisme : polymorphisme de compilation (surcharge de méthodes) et polymorphisme d'exécution (sous-typage).

    Exemple :

    class Animal {
      void faireDuBruit() { System.out.println("Bruitage"); }
    }
    class Chien extends Animal {
      void faireDuBruit() { System.out.println("Aboie"); }
    }

    5. Abstraction

    L'abstraction permet de définir des classes et des méthodes abstraites pour ne garder que les éléments essentiels et masquer les détails d'implémentation. Les classes abstraites ne peuvent pas être instanciées directement et sont destinées à être étendues par d'autres classes.

    Exemple :

    abstract class Forme {
      abstract double aire();
    }
    class Cercle extends Forme {
      double aire() { return Math.PI * rayon * rayon; }
    }

    Avantages de la POO

    • Réutilisabilité du code : Grâce à l'héritage et aux classes réutilisables.
    • Modularité : Les objets sont indépendants, facilitant la maintenance et la lisibilité.
    • Encapsulation : Protection des données et réduction des erreurs.
    • Facilité de débogage : Structure plus claire et organisée.

    Résumé

    La Programmation Orientée Objet est un paradigme puissant qui organise le code en objets, facilitant la réutilisation, la maintenance et la modularité. Les quatre concepts clés (encapsulation, héritage, polymorphisme, abstraction) permettent de construire des applications évolutives et maintenables.

    Exemple : Gestion d'un Restaurant avec la POO

    Dans cet exemple, nous allons modéliser la gestion d'un restaurant en utilisant les concepts de la POO. Nous allons créer des classes pour les différents types de produits alimentaires et de boissons, et utiliser les concepts d'héritage, d'abstraction, d'encapsulation et de polymorphisme pour gérer les commandes.

    Diagramme de Classes

    Voici un schéma simplifié des classes du restaurant :

    • Food (interface) : Définie avec les méthodes getPrice() et getCalories().
    • Drink (classe abstraite) : Implemente Food, possède un attribut aCan et est étendue par les classes Coke, AppleSmoothie.
    • Bread et Sandwich (classes abstraites) : Implemente Food. Bread est étendue par FrenchBaguette, et Sandwich par HamSandwich.
    • CustomerOrder : Gère les commandes en utilisant une liste de produits Food.

    Code Java pour l'Exemple du Restaurant

    interface Food {
      float getPrice();
      int getCalories();
    }

    abstract class Drink implements Food {
      protected boolean aCan;
      public boolean isACan() { return aCan; }
    }

    class Coke extends Drink {
      private float price = 1.20f;
      private int calories = 105;
      public float getPrice() { return price; }
      public int getCalories() { return calories; }
    }

    abstract class Bread implements Food {
      protected float price;
    }

    class FrenchBaguette extends Bread {
      private float price = 0.80f;
      private int calories = 700;
      public float getPrice() { return price; }
      public int getCalories() { return calories; }
    }

    import java.util.ArrayList;
    import java.util.List;

    class CustomerOrder {
      private List<Food> orderItems = new ArrayList<>();
      public void addItem(Food item) { orderItems.add(item); }
      public float getTotalPrice() {
        float total = 0;
        for (Food item : orderItems) { total += item.getPrice(); }
        return total;
      }
    }

    public class Main {
      public static void main(String[] args) {
        CustomerOrder order = new CustomerOrder();
        Food coke = new Coke();
        Food baguette = new FrenchBaguette();
        order.addItem(coke);
        order.addItem(baguette);
        System.out.println("Total Price: " + order.getTotalPrice());
      }
    }
